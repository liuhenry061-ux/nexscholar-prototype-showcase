US201 focused experience

Guide new users with actions, not long instructions.

The live platform already shows signs of tutorial support, but the project charter correctly identified that first-time onboarding needs to be more interactive. This page turns onboarding into a short role-aware journey that helps users get to value faster.

Onboarding goal

End the first-use flow with one useful next step, not a passive confirmation message.

Four-step student onboarding

This structure replaces text-heavy instructions with a simple path: define goal, complete profile, try a guided example, then take one meaningful action.

1
Select your goal

Choose between finding a supervisor, screening opportunities, or building a research profile.

2
Complete key profile fields

Highlight only the fields that improve matching accuracy instead of overwhelming users with the full account setup.

3
Run a guided example

Show one sample search, one explanation panel, and one compare step to teach the platform through use.

4
Take the first next action

The tutorial should end by moving the user into the real product flow, not by stopping at an information page.

Support cards around the journey

Small support panels reduce uncertainty and tell the user why each step matters.

Profile readiness

Show a visible readiness score and tell the user which missing item matters most for stronger matching.

Guided example

Walk through one realistic scenario based on the user's chosen field instead of generic help text.

Confidence note

Clarify what the system knows and what it is inferring so the user can trust the recommendations.

Sprint 3 return-use improvements

The onboarding page now does more than help users begin. It also shows why they should come back later and what progress they can continue.

After onboarding

Users should transition into a live platform state such as comparing recommendations, improving profile readiness, or saving a future opportunity.

Before the next visit

The system should leave the user with a remembered next action, which supports the Sprint 3 goal of stronger ongoing engagement.

Why this matters

Less friction

New users see the shortest useful path instead of a wall of instructions.

Better quality input

Users are nudged to complete the profile fields that actually affect matching quality.

Faster activation

The first useful action happens during onboarding, not after it.
